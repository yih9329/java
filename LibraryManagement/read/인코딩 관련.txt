가장 먼저 확인할 사항: 자바 프로그램의 인코딩형식, 웹페이지의 인코딩 형식, 데이터베이스의 인코딩 형식


1. html 및 jsp에서 request로 넘어오는 한글은 8859_1 인코딩형식으로 넘어온다.
2. 보통 자바프로그램은 유니코드를 사용한다(EUC-KR, MS949, UTF-8)
3. 자바 프로그램에서 한글을 제대로 받기 위해서는 new String(str.getBytes("8859_1"), "UTF-8")과 같이
   인코딩을 해주어야 깨지지 않고 나오게 된다.
4. request 객체로 파라메타를 받을 시에는 작업의 시작부분에 request.setCharacterEncoding("UTF-8")을
   적어준 후 파라메타를 받으면 한글이 깨지지 않는다.
5. Command 객체를 사용할 시에는 멤버변수의 값에 8859_1 인코딩 형식의 데이터가 들어가있으므로 Command 객체
   setter에 3번 코드를 넣어주면 원하는 인코딩형식의 데이터가 들어간다.
6. view 페이지의 인코딩 형식과 Command 객체에서의 인코딩형식이 같다면 멤버객체의 값을 깨짐없이 읽을 수 있다.
7. 데이터베이스의 인코딩형식에 맞는 데이터를 자바프로그램에서 인코딩해준 후 보내줘야한다.
   (db의 형식이 utf-8이고, 입력할 데이터의 형식이 utf-8이라면 이상없이 들어간다. 하지만 둘의 형식이 다르다면
    별도의 가공을 통해 전달을 해줘야한다.)

** 느낀점
이왕이면 자바 프로그램, db, 웹페이지의 인코딩 형식을 모두 utf-8로 설정해주고 프로젝트를 진행한다.
그러면 http request로 오는 인코딩형식이 8859_1인 것만 주의하여 커맨드객체 및 getparameter를 사용할 때
인코딩만 잘해주면 문제없이 한글을 쓸 수 있다.
